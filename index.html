什么是存储器？

存储器是计算机的记忆部件，用来存放程序及所涉及的数据。主存储器存放当前正在执行的程序和使用的数据，可以直接被CPU存取。

存储容量是指存储器所具有的存储单元个数，其基本单位是字节(B),常用的单位有KB、MB、GB、TB

1KB=2^10B-1024B,1MB=2^20B,1GB=2^30B 1TB=2^40B

什么是存储器地址？（重点）

主存储器由大量存储单元组成。为了区别每个单元，我们将他们编号，于是每个存储单元就有了存储器地址（Address）。•存储器地址就是存储器中存储单元的编号（编号＝地址）

每个存储单元存放1字节的数据。1字节（byte B）包含8个二进制位（bit b）

•采用十六进制数来表达地址

•Intel 8086具有1兆字节（1MB）存储器容量

•存储器地址表示为：00000H ～ FFFFFH

•其中大写H（或小写h）表示是十六进制数

注：二进制后缀字母为B/b，十进制后缀字母为D/d，十六进制后缀字母为H/h。

关于十六进制：

由于二进制数书写较长，难以辨认，因此常用易于与之转换的十六进制数来描述二进制数。

字符的编码

由于字节为计算机的基本储存单位，所以常以8个二进制位为单位表达字符

什么是BCD？

Binary Coded Decimal（二进制编码的十进制数），即一个十进制数位（0-9）在计算机中用4位二进制编码表示。常用的BCD码是8421BCD码，即用4位二进制编码的低19个编码表示0-9这十个数字，如图。

例：0100 1001 0111 1000.0001 0100 1001 十进制真值：4978.149（上图对应）

00111010B=3AH F2H=11110010B

10H=00010000B=16D ABH=10101011B=171D

压缩BCD码：一个字节表达两位BCD码 10000111B（87H)

非压缩BCD码：一个字节表达一位BCD码（低4位表达数值，高4位常设置为0）

00001000 00000111B（0807H）

什么是ASCII？

ASCII(American Standard Code for Information Interchange)，即美国标准信息交换码

•标准ASCII码用7位二进制编码，有128个，由于计算机存储单位为8位，故表达时最高位通常为0.

常用ASCII码（重要）

回车：ODH 换行 OAH 空格 20H 响铃 07H

•数码0～9：30H～39H

•大写字母A～Z：41H～5AH

•小写字母a～z：61H～7AH

什么是原码 反码 和 补码？

最高有效位表示符号（正数为0，负数为1），

正数的反码与原码相同，负数的反码则为它的正数各位（包括符号位）按位取反形成。

例：X=105=01101001B [X]反=01101001B

​ X=-105 =11101001B [X]反=10010110B

补码中最高位表示符号，有符号整数在计算机中默认采用补码

正数补码直接表示数值大小(与原码和反码相同)

负数补码：将对应正数补码取反加1

[105]补码＝01101001B

[-105]补码＝[01101001B]取反＋1＝10010110B＋1＝10010111B

负数求真值：

补码：11100000B

真值：-([11100000]求反＋1)＝-(00011111+1)＝-00100000＝-2^5＝-32

由于符号要占用一个数位，8位二进制补码中只有7个数位表达数值，表示的数值范围：-128～+127

对于n位二进制编码范围为：-2^n -1~2^(n-1) -1

什么是寄存器？

寄存器是CPU内部的高速存储单元，它们为处理器提供各种操作所需要的数据或地址等信息

•汇编语言程序采用它们各自的符号名（重点）

8086的寄存器有哪些？

8086的寄存器组分成8个通用寄存器，4个段寄存器，1个标志寄存器和1个指令指针寄存器，他们均为16位。

1.通用寄存器

(1)数据寄存器

8086有4个16位数据寄存器：AX BX CX DX，他们都还可以分成高8位和低8位两个独立的寄存器(AH/AL)，且对其中某8位的操作，并不影响另外对应8位的数据

(2)变址寄存器

变址寄存器常通过改变寄存器表达的地址指向数组元素，用于存储器寻址时提供地址。

•SI (Source Index) 是源变址寄存器 •DI (Destination Index) 是目的变址寄存器

(3)指针寄存器

指针寄存器用于寻址内存堆栈堆栈内的数据

BP和SP是（堆栈）指针寄存器，用于指向堆栈中的数据。

其中，SP是堆栈指针寄存器（指向堆栈栈顶的指针），指示栈顶的偏移地址。它会随着处理器处理有关指令自动增大或减小，所以SP不应该再用于其他目的，实际上可归类为专用寄存器。

BP为基址指针寄存器（指向堆栈某处的指针），表示数据在堆栈段中的基地址。

SP和BP寄存器与SS段寄存器联合使用确定堆栈段中的存储单元地址

2.标志寄存器（FLAGS）

标志（Flag）用于反映指令执行结果或控制指令执行形式。

8086中，各种常用的标志形成16位标志寄存器FLAGS（程序状态字PSW寄存器）分成两类：

(1) 状态标志

状态标志是最基本的标志，用来记录指令执行结果的辅助信息。加、减运算和逻辑运算指令是主要设置他们的指令，有些其他指令的执行也会相应地设置它们。

①进位标志CF（Carry Flag）

当运算结果的最高有效位有进位（加法）或借位（减法）时，进位标志置1，即CF = 1；否则CF = 0。

例：AAH + 7CH＝（1）26H，有进位：CF = 1

奇偶标志PF（Parity Flag）

调整标志AF（Adjust Flag）

零标志ZF（Zero Flag）

符号标志SF（Sign Flag）

溢出标志OF（Overflow Flag）

(2) 控制标志

控制标志可由程序根据需要用指令设置，用于控制处理器执行指令的方式

方向标志DF（Direction Flag）仅用于串操作指令中。

中断允许标志IF（Interruptenable Flag）简称中断标志。

陷阱标志TF（Trap Flag），常称为单步标志。

3.指令指针寄存器（Instruction Pointer）

IP是专用寄存器，具有自动增量的能力。处理器执行完一条指令，IP中的值就加上该指令的字节数，从而指向下一条指令。即：指示代码段中指令的偏移地址。

•它与代码段寄存器CS联用，确定下一条指令的物理地址

•计算机通过CS : IP寄存器来控制指令序列的执行流程

4.段寄存器

•8086有4个16位段寄存器，每个段寄存器确定一个逻辑段的起始地址，每种逻辑段均有各自的用途

•CS（Code Segment）

•指明代码段的起始地址

•SS（Stack Segment）

•指明堆栈段的起始地址

•DS（Data Segment）

•指明数据段的起始地址

•ES（Extra Segment）

•指明附加段的起始地址

数据的存储格式

8086的字长为16位，由2字节组成，称为字（Word）D7-D0 双字为4个字节（32位）D31-D0

存储器以字节为单位存储信息，每个存储单元的编号被称为存储器地址。
